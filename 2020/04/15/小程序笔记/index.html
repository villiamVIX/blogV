import导入/include导入

  • import导入主要用于导入<template>的不支持递归(套娃导入)

  • include用于导入共同的页头部和页尾部，可以递归导入；不能导入wxs、template的组件

  • 冒泡：从内->到外

组件

  • 写好子组件，在爹组件中的json文件里useingComponents：里注册好，若多出要用就直接在app中全局注册

  • 子组件不能不用id、属性、标签/只能用class选择器

  • 默认父子组件样式隔离互不影响；若想影响在js文件中的component的options的styleIsolation:属性变为apply-shared

  • 组件间样式 externalClasses

1.在子组件定义好externalClasses属性用数组
2.在爹组件写好class样式
3.在爹组件中<dad 订好的样式名=’爹中样式’>

  • 父传子传值 properties

1.在组件Component({})中写好接受的值名
2.默认用value：’’
3.可以监听用 observer：跟vue的watch一样接受（new,old）然后方法

  • 子传父传值 用方法triggerEvent

  • 爹找组件的方法selectComponent
    用这个方法可以直接改子里的数据，但是最好子暴露出子组件内部的数据修改方法，供爹调用

  • 子组件的方法只能写在method里

迁移VUE的icomoon的svg样式

  • 因为小程序支持用BASE64显示，那就转base64在看
    网上看了别人的教程 都说直接转码替换@font-face事实我用的时候不行。
    试了4个小时，转码无数遍，准备放弃，去iconfont重新挑选导入的时候，试着原生导入方法再改一遍！成了
    ！必须原style.css的@font-face齐聚一堂 仅替换.ttf

  1. 和vue一样，先去阿里iconfont下好想要的svg图片(建议本地备份一遍/iconfont里项目文件夹备份一份)

  2. 导入icomoon后-下载好svg压缩包(下称1号文件)=>有用的(font/.ttf)+style.css

  3. vue中是直接使用；小程序要提出1中的font/.ttf文件去下述转码网址转换 =>得到stylesheet.css

  4. 转码后把1里的style.css的@font-face里的后缀是format(‘truetype’)的文件 => 替换为2中下载的@font-face里的那行truetype

  5. 爱怎么导入/还是直接写app.wxss都可以 ！page是可以直接用

  6. 在子组件中本身是获取不到app的全局导入的图标样式

  7. 需要在子组件中的 .js 开启 options: {
    addGlobalClass: true,
    },表示接受全局样式 / 因为单独在子组件导入样式是不显示也不现实的。

小程序登录流程

  • 前端获取code(5分钟失效)传给node服务器

  • node将code+appid+appsecret发给微信官方接口换取session_key+openid(wx用户唯一标识)

  • node服务器判断数据库有无用户=>分情况登录=>将状态返回给小程序前端
